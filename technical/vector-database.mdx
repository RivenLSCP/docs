## Overview

Sera's Vector Database is built on ChromaDB technology, providing a high-performance, distributed solution for storing and querying vector embeddings. It's designed specifically for AI applications with features like real-time indexing, efficient similarity search, and automatic synchronization.

### Basic Usage

```python
from sera.vector import VectorDB

# Initialize client
db = VectorDB.connect(
    cluster="production",
    api_key="your_api_key"
)

# Create collection
collection = db.create_collection(
    name="embeddings",
    dimension=768
)

# Add vectors
collection.add(
    vectors=embeddings,
    metadata=metadata,
    ids=unique_ids
)

# Query
results = collection.query(
    vector=query_vector,
    top_k=10
)
```

### Query Operations

```python
# Complex query with filters
results = collection.query(
    vector=query_vector,
    filter={
        "category": {"$in": ["tech", "science"]},
        "confidence": {"$gt": 0.8}
    },
    top_k=10,
    include_metadata=True
)

# Batch query
results = collection.batch_query(
    vectors=query_vectors,
    top_k=5,
    batch_size=100
)

# Streaming query
async for result in collection.stream_query(query_vector):
    process_result(result)
```

## Data Management

### Backup & Recovery

```python
# Create backup
backup_id = db.create_backup(
    collections=["embeddings"],
    destination="s3://backup-bucket"
)

# Restore from backup
db.restore_backup(
    backup_id=backup_id,
    target_cluster="new-cluster"
)
```
